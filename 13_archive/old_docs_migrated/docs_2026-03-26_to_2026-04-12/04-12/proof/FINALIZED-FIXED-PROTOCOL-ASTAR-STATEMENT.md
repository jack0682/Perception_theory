## 1. Purpose

This note merges the event and horizon pieces into one compact theorem-facing statement of the fixed-protocol accessibility surrogate.

---

## 2. Finalized Compact Definition

For a named protocol `S`, define

```text
A_S(G, lambda; U_B)
:= P( Enter_S(G, lambda; U_B) before the finite termination of S ).
```

Equivalently, `A_S(G, lambda; U_B)` is the probability that the trajectory generated by protocol `S`, under its own declared finite run specification, first enters the theorem-facing local neighborhood `U_B(lambda)` before that run terminates.

---

## 3. What This Definition Includes

This compact definition already packages together:

1. a **protocol tag** (`S`),
2. a **target family neighborhood** (`U_B(lambda)`),
3. a **first-hit event** (`Enter_S`),
4. and a **finite protocol-native horizon** (termination rule of `S`).

So it is the smallest complete accessibility object currently justified by the evidence.

---

## 4. Why This Is the Right Current Endpoint

This definition is strong enough because:
- it yields a meaningful quantity for protocol comparison,
- it matches how the current experiments are actually run,
- it plugs directly into the gap object
  `Gap_{raw,seed}(G, lambda; U_B) = A_seed - A_raw`.

It remains weak enough because:
- it does not claim search-neutrality,
- it does not require infinite-time control,
- and it does not pretend that basin volume is already analytically characterized.

---

## 5. Safe Current Interpretation

The strongest current theorem-support reading is:

> under a named finite protocol, accessibility means probability of first entering the theorem-facing local target neighborhood before that protocol terminates.

This is a complete fixed-protocol surrogate, though still not a theorem-level bound.

---

## 6. What Remains Open

The next unresolved refinement questions are now genuinely narrower:

1. should the event remain **first-hit**, or be strengthened to **stable-entry**;
2. should different protocols eventually be normalized to a common horizon notion;
3. how can one turn empirical accessibility rates into theorem-level upper/lower bounds.

---

## 7. Next Trigger

The next theorem-facing choice should compare:

1. strengthening the event from first-hit to stable-entry, or
2. strengthening the finite horizon from protocol-native to a more comparable normalized stopping notion.
