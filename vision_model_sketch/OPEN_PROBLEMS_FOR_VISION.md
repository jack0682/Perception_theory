이 문서는 두 종류의 OP를 다룬다:

- **비전 OP (V-x)**: 비전 응용에서 *새로 드러나는* 문제. 이론 자체는 우회 가능하지만 비전 모델은 우회 못 함.

## 1. Critical Blockers 🔴 (이게 안 풀리면 시작도 안 됨)

### 1.1 OP-0005: K-Selection 메커니즘 (이론 OP)
**상태**: HIGH severity, OPEN. σ-framework + CN15가 *부분* 답 제공.

**진술**: 응집장이 *몇 개의* 형태로 분해되는지 결정하는 메커니즘이 없다.
- 외부 고정? (현재 v1.3 가정 — A-0012)
- 에너지 최소화에서 출현? (M-1으로 부정됨)
- 모델 선택 (BIC, free energy)?
- 운동학적 결정 (메타안정 장벽)?

**비전 영향**: **치명적**.
- 이미지마다 객체 수가 다르다. K를 hyperparameter로 두면 일반화 불가.
- "이 이미지엔 사물이 *몇 개*"라는 가장 기본 질문에 답 못 함.
- Slot Attention이 K 슬롯을 고정값으로 두는 이유 — 그들도 못 풀었기 때문.

**필요한 진전**:
- σ-jump formalization (NQ-148) — 부분 답
- 동적 K (변동 가능): 형태 birth/death 정리. T-Birth-Parametric은 *균질→비균질*의 출현만 다룸. K → K+1 전이는 미해결.
- Static/Dynamic Separation (CN15)을 K-selection 알고리즘으로 변환하는 길.

**우회 가능성**: 부분적. 고정 K_max로 시작해서 일부 슬롯이 비활성화되도록 학습. 단, 이건 *구현적 우회*이고 이론적 답이 아님.

---

### 1.2 다중 형태 σ-Framework 확장 (Phase 5) (이론 OP, MO-1 재활성화)
**상태**: 단일 형태 σ는 W4에서 닫힘 (Cat A). 다중 형태는 미해결.

**진술**: σ-signature $\sigma(u^*) = (\mathcal{F}; \{(n_k, [\rho_k], \lambda_k)\})$가 단일 형태 ($\Sigma_m$, smooth manifold) 위에선 잘 정의됨. 다중 형태로 확장하면 $\Sigma^K_M$ — manifold with corners — 이 되어 **Morse theory가 직접 적용 안 됨 (MO-1)**.

**비전 영향**: **치명적** (다중 객체 다룰 때).
- 한 이미지에 여러 객체 → 다중 형태 필요 → Phase 5 필요.
- 단일 형태만 다루면 이미지당 한 객체만 처리 가능 (CLEVR 단순 버전).
- 비전 모델이 진짜 작동하려면 반드시 풀려야 함.

**필요한 진전**:
- Stratified Morse on $\Sigma^K_M$ — corners를 다루는 Morse 변종.
- Multi-formation σ-irrep 분류표.
- $K$-field 사이의 σ-coupling 정리.

**우회 가능성**: 어렵다. K-field architecture로 에너지 차원에선 분해 가능하지만, *형태 사이 상호작용*의 정리는 못 우회.

### 1.3 V-1: 픽셀-그래프 다리 (비전 OP)
**상태**: NEW. 이론 OP에 없음 (이론은 그래프에서만 작동).

**진술**: 이론은 *유한 그래프* 위에서 작동. 비전은 *연속 이미지*. 픽셀 → 그래프 노드 매핑이 명시되지 않음.

세부:
- 한 픽셀 = 한 노드? (10^6 노드 — 스케일 문제)
- 한 패치 (예: 16×16) = 한 노드? (정보 손실, ViT 토큰화와 동일)
- 적응적 슈퍼픽셀? (학습 가능)
- 백본 특징맵의 한 위치 = 한 노드? (가장 자연스러움)

**비전 영향**: 치명적. 이게 정해지지 않으면 모델 입력 인터페이스 자체가 미정.

**필요한 진전**:
- OP-0022 (연속 극한) 이론과 연결.
- 실험적: 다양한 매핑의 성능 비교.
- 이론적: 연속 극한에서 SCC가 어떻게 정의되는지.

**우회 가능성**: 가능. ViT 토큰을 노드로 쓰고 토큰 사이 attention을 인접 커널로 쓰면 즉시 작동. 다만 이론적 정당성은 약함.

## 2. High Priority 🟠 (모델 일반화·안정성)

### 2.1 OP-0006: 분기 선택 이론 (이론 OP)
**상태**: HIGH severity. Type A/B 분류 retracted. 대안 없음.

**진술**: 같은 매개변수에서 여러 안정 형태 (branches)가 존재 가능. 어느 branch에 도달할지 결정하는 *선택 규칙*이 없음.

**비전 영향**: **높음**.
- 같은 이미지 → 학습 시드에 따라 다른 분해 결과.
- 재현성 문제. (학습마다 결과가 다름)
- 4차원 점수가 *어느 branch에서 측정한 것인지* 불명.

**필요한 진전**:
- IC 분포에서 branch probability 계산.
- T-PreObj-1 (v) IC-protocol dichotomy를 일반화.
- Branch transition graph.

**우회 가능성**: 부분적. 고정 IC + 결정론적 학습으로 재현성 확보. 일반 분기 선택은 미해결.

---

### 2.2 OP-0011: 운반 커널 유일성 (이론 OP)
**상태**: MEDIUM severity. exp30–35 진행 중.

**진술**: 현재 운반 커널 (entropy-regularized OT)은 axiom E1–E5를 만족하는 *한 가지* 실현. 다른 실현이 가능한지 미해결.

**비전 영향**: **중간~높음**.
- 비디오 객체 추적 시 *다른 운반 형태*가 다른 ID 매칭을 줄 수 있음.
- T-Persist-1(e) 집중성 보장은 *현 커널에 한정*.

**필요한 진전**:
- E1–E5를 만족하는 커널 가족의 분류.
- 그 중 어느 것이 *수렴 결과 동일*한지.

**우회 가능성**: 가능. 한 커널을 고정하고 그 위에서 만들면 됨. 다만 "이 모델은 본질적이다"라고 주장 못 함.

---

### 2.3 V-2: 미분가능 형태 측정 $\mathcal{Q}_{\mathrm{morph}}$ (비전 OP / 부분 엔지니어링)
**상태**: NEW. 이론에선 정의 (CN12, persistence homology) 명확하지만 GPU 미분 가능 구현이 어려움.

**진술**: $\mathcal{Q}_{\mathrm{morph}} = \frac{\ell_{\max} - c}{1-c} \cdot \mathrm{Artic}$ 는 superlevel-set persistence diagram의 bar 길이 기반. GPU에서 미분 가능하게 구현하기 어려움.

**비전 영향**: **높음** (Inside 점수 계산에 필수).
- 라이브러리 (`gudhi`, `ripser`)는 CPU 전용.
- Differentiable persistent homology 연구가 있지만 (Carrière et al., 2021) 통합 비용 큼.

**필요한 진전**:
- 미분 가능 persistence layer 구현.
- 또는: $\mathcal{Q}_{\mathrm{morph}}$의 *근사*를 찾되 QM1–4를 만족하는 것.

**우회 가능성**: 가능. Smoothness + double-well penalty로 *간접* 측정 (E_bd만 쓰고 Inside 명시적 출력 안 함). 다만 4차원 진단 벡터가 3차원이 됨.

---

### 2.4 V-3: 가변 K across batch (비전 OP, OP-0005와 결합)
**상태**: NEW. OP-0005의 비전 특화 형태.

**진술**: Batch 내 이미지마다 객체 수가 다름. 모델 출력의 *모양*이 가변적.

**비전 영향**: **높음**.
- PyTorch tensor는 고정 dim 선호.
- DETR/Slot Attention 모두 고정 K_max로 우회 (낭비 슬롯).

**필요한 진전**:
- OP-0005 해결의 *PyTorch 친화적* 형태.
- Sparse 표현 또는 dynamic computation graph.

**우회 가능성**: 엔지니어링 우회 (K_max + masking). 이론적 우회는 OP-0005 의존.

**상태**: τ=1/2만 Cat A. General τ는 Cat C.

**비전 영향**: 중간. Multi-scale 분석 (객체 vs 부분-객체 vs 부분-부분) 시 다른 τ 필요.

**우회**: τ=1/2 고정. 단일 스케일 모델은 OK.

**우회**: 매 프레임 쌍 단위로 처리 (T-Persist-1만 사용). 누적 오차는 학습으로 보완.

### 3.4 OP-0021: 확률 동역학 (이론 OP)
**상태**: LOW severity in `open_problems.md`, 비전에선 MEDIUM.

**진술**: 이론은 결정론적 gradient flow 중심. 노이즈 하 동역학 미정립.

**비전 영향**: 중간. 자연 이미지엔 노이즈가 항상 있음. 모델의 노이즈 강건성 보장 약함.

**우회**: 학습 시 augmentation으로 noise robustness 흡수. 다만 *정리*는 못 줌.

## 4. Engineering Issues ⚙️ (이론 OP 아님, 그러나 구현 block)

### 4.1 GPU 친화적 미분 가능 폐포
- `scc/operators.py`는 numpy + 명시적 JVP.
- PyTorch autograd로 포팅 필요.
- DEQ 스타일 implicit differentiation 적용.
- **추정 작업량**: 1~2개월.

### 4.2 100→10^4~10^6 노드 스케일링
- 현재 테스트는 10×10 ~ 32×32 그리드.
- 비전은 224×224 (50K 노드) ~ 512×512 (260K 노드).
- 폐포·분리 모두 메시지 패싱 → linear scaling. 가능.
- 메모리: K개 형태 × 노드 수 → batch 시 GPU memory critical.
- **추정 작업량**: 2~3개월 (sparse ops + 효율화).

### 4.3 백본 통합
- 어떤 백본? (DINO v2, SAM encoder, CLIP-ViT)
- Frozen vs fine-tuning?
- 백본 출력의 차원과 SCC 응집장의 관계.
- **추정 작업량**: 2~3개월 (실험적).

### 4.4 학습 안정성
- $a_{\mathrm{cl}}$이 4를 넘으면 수축 깨짐 → 학습 발산.
- $\lambda$ 가중치들의 균형 (5+ hyperparameter).
- 평형 모듈이 수렴 안 하면 gradient 계산 실패.
- **추정 작업량**: 진행 중 계속.

### 4.5 평가 프로토콜
- "비지도 객체 발견"의 표준 벤치마크 부재.
- CLEVR, MOVi 외에 자연 이미지 OOD 평가 어려움.
- 4차원 점수 *해석*: 어느 점수가 어느 작업에 중요?
- **추정 작업량**: 1~2개월 (평가 셋업).

## 8. 핵심 결정 포인트

이 디렉토리를 다시 들여다볼 때 검토할 *결정 포인트* 3개:

### 결정 1: OP-0005 (K-selection)에 진전이 있는가?
- ✅ Yes → 본격 다중 객체 모델 시작 가능
- ❌ No → 단일 형태 모델까지만, 또는 K_max 임시방편

### 결정 2: Phase 5 (다중 형태 σ)가 시작되었는가?
- ✅ Started → 1.5년 내 다중 객체 vision model 가능성
- ❌ Not started → 단일 형태에서 멈춤

### 결정 3: V-1 (픽셀-그래프 다리)에 대한 표준이 정해졌는가?
- ✅ Yes → 모델 인터페이스 통일 가능
- ❌ No → 각 실험마다 다른 매핑, 비교 어려움

## 9. 이 OP들이 비전이 *아닌* 다른 응용에서도 중요한가?

흥미로운 부수 사실:

| OP | 비전 외 응용 영향 |
|----|------------------|
| OP-0005 K-selection | 인지과학 (객체 인식), 분자생물학 (도메인 분해), 사회과학 (커뮤니티 검출) — *모든 곳에서 중요* |
| Phase 5 multi-σ | 응집물리 (다중 모드 페이즈), 양자장론 (다중 vacuum) |
| OP-0011 transport | 발달생물학 (cell lineage), 경제학 (자원 흐름) |
| OP-0006 branch | 진화론 (적응 분기), 학습이론 |
| OP-0022 continuous limit | 수학적 일반화의 핵심 |

**즉, 비전 응용을 위한 OP 해결은 다른 응용도 동시에 풀어줌.** 비전이 가장 가까운 첫 응용일 뿐.

## 10. 이 문서의 활용

- **이론 작업 시**: 비전 응용을 *직접* 신경 쓰지 않되, 위 OP들이 풀리면 부수적으로 비전이 열린다는 인식.
- **응용 시작 시**: 어느 OP가 풀렸는지 먼저 확인. Critical 둘 (OP-0005, Phase 5) 닫히기 전엔 본격 시작 자제.
- **공저자/논의 시**: 이 문서가 "왜 지금 안 만드나"의 답.
