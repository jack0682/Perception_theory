## §2. New Conceptual Hazards in Redirection Mode

기존 Day 3 v1 의 hazards (H1' confidence inflation recursive / H2' scope creep doubled / H5' pacing dissonance compounded) 는 deliberate-taper 가 폐기됨에 따라 **재구성**되어야 한다.

**(H6) Redirection without commitment.** Day 3 가 OP recalibration + 4 starts 로 표면적 계획만 만들고 실제 substantive work 0 인 채로 끝나면, 이는 "검토 후 또 다른 메타-결정 cycle" 이 됨 (W4 debt 가 더 커짐). 새 calendar 를 만드는 것 자체가 substantive 가 아니다 — calendar 는 가이드일 뿐. 실제 OP-0009-Pre / NQ-G1-2-ext / OP-0005 어디든 한 곳의 *손에 잡히는 진전* 이 있어야 한다.

**(H7) Cherry-picking the easiest debt.** W2 (NQ-G1-2-ext production reach) 는 ~1-2h cheap; W1 (OP-0009-Pre) 은 weeks-of-work substantial. cheap-easy-but-low-leverage 만 처리하고 expensive-but-high-leverage 를 다시 미루면, 누적 부채는 다시 자란다. Day 3 는 *high-leverage debt* (W1 + W3) 에 substantive 진전을 우선해야 한다 — 비록 그것이 ~2h substantive 진전 + ~1h 메타-recalibration 이 되더라도.

**(H8) Recalibration as substitute for action.** "W7-W10 calendar 를 다시 작성했다" 는 자체가 substantive 진전인 양 착각하지 말 것. Calendar 는 ~30 min 작업; 실제 OP-0009-Pre 의 mathematical work, P-F framework 의 axiom proposal, NQ-G1-2-ext 의 design sketch 가 substantive. Day 3 산출물 평가 기준: *"이 산출물이 W7+ 의 substantive work 의 시작점이 되는가, 아니면 또 다른 plan/proposal layer 인가?"*

**(H9) Closure-week boundary 깨짐.** Day 3 가 OP-0009-Pre 의 *actual mathematical work* 을 시작한다는 것은 closure-week commitment (Option B / W6 strategic plan §7) 와 모순될 수 있다. W7-week 에 본격 시작해야 할 것을 W6 안에서 시작하면 W6 가 expansion week 이 되는 risk. Mitigation: Day 3 substantive work 는 *starting point* 만 (~1.5-2h), 본격 진행은 W7 으로. "W6 = closure + W7 redirection 의 starting line" 이 되도록.

## §3. Mental Models for Day 3 Shape (v2)

- (R1-c, ~30-45 min) **P-F framework escalation proposal** — P-F 를 OP-0010-0013 Medium 에서 OP-0005-adjacent High 로 승격하는 정당화 + W7-W10 어디에 위치하는지. 산출: `03_pf_framework_escalation_proposal.md`.

Time: ~3.5-4h. Risk: moderate (R1-b 가 OP-0009-Pre의 mathematically 어려운 영역 — 안 풀릴 가능성). Value: very high — 누적 부채 4개 모두에 substantive 시작점 제공.

**(R3) Recalibration-only day** (~1-1.5h, 최소). R1-a 만 + Day 4 plan. W7-W10 calendar 가 가장 high-leverage 이라고 판단되면 그것만 깊이 + 나머지 다 reschedule.

**(R-anti-1) Skeleton refinement day** (v1 plan). OP recalibration 없이 v1 의 skeleton refinement + W7 Stage 1 dispatch skeleton 만. **명시적으로 폐기** — 이는 administrative work 이고 누적 부채 4개를 더 키운다.

**(R-anti-2) Unfocused multi-start day**. 4 substantive goals 를 모두 시작하지만 어느 것도 substantive starting point 에 도달 못하는 case (H8 hazard). Day 3 산출물 0개 substantive starting point 이면 redirection 자체가 무의미.

자연스러운 Day 3 plan 은 (R1) primary, (R2) fallback, (R3) 최소. v1 의 skeleton refinement + W7 Stage 1 dispatch 는 W6 D5-D6 으로 reschedule.

## §5. Reflection on the Whole 6-Week Pattern

**(O4') Closure cycles 가 expansion cycles 를 늦춘다.** W4 04-12-04-18 Research OS 시도/포기 → W5 σ-framework canonical merge → W6 G1-G4 closure week. 매 cycle 마다 새로운 "정리 layer" 가 추가되었지만, OP-0005 / 0008 / 0009 의 substantive 진전 비율은 줄어들었다. W6 EOD 시점 OP-0005 K-Selection 4-layer 중 1 layer (Layer D = Commitment 16) 만 RESOLVED, 3 layers (A/B/C) working. 동일 pattern 이 W7-W10 에 반복되면 v2.0 (W11-W12) 시점에 OP-0009-Pre 만이 아니라 OP-0005 / 0008 도 미해결.

**(O5') Documentation의 자기-증식 (self-proliferation) 패턴 식별.** W4 reformulation_plan.md → W5 σ-framework supporting structures → W6 parking-lot inventory + 14 addendums + 12 NQ resolution + Issue #1-#5 series. 각각 substantive starting point 에서 출발해 *closure documentation* 으로 자기-증식. working/MF/ 50개 파일 + working/SF/ 10+개 파일 + working/CE/ 등은 모두 "원본 작업 + 그 작업의 정리/감사/패키징" 의 누적. **이 자기-증식은 H8 hazard ("recalibration as substitute for action") 의 누적 형태**.

**(O6') 진정한 substantive theorem creation 의 빈도.** 최근 2달 (W4-W6, ~6주) 의 새 Cat A: T-PreObj-1 + T-PreObj-1G (W4), T-V5b-T (W4 extended), T-σ-Lemma-1/2/3 + T-σ-Theorem-3/4 (W5 D1), T-Commitment-14-Multi-Static + T-σ-multi-A-Static + T-σ-multi-D-Static (W5 D4), T-L1-F (W5 D6), T-L1-M (W6 D1 EOD). ~12 theorems / 6주 = ~2/주. 그러나 W6 의 marginal 새 theorem = 1 (T-L1-M, working draft 가 W5 D7 에 있던 것이므로 W6 의 *순수 새 작업* 은 0). W7-W10 가 같은 pattern (서비스/감사/closure 가 substantive theorem creation 보다 우세) 이면, *이론의 진전이 멈춘다*.

**(O7') 가장 굵은 trend: ontological commitment 와 mathematical work 사이 거리 증가.** Commitment 1 (u_t primitive) + Commitment 11 (crisp objects derivative) 이 명시적으로 stated 된 이후에도, *작업의 mathematical entity 들* 은 점점 K-field, K_act, K_soft, K_bar 같은 *count-indexed* 객체. SCC 의 가장 큰 ontological promise 는 *"객체 이전" cohesion 의 mathematical theory* 인데, 작업의 80%+ 가 객체-이후 (slot, count, label) 의 mathematical theory. Day 3 redirection 의 가장 깊은 motive 는 이 거리를 **명시적으로 확인 + 일부 좁히기 시작** 하는 것.

## §6. What Could Go Wrong (v2 — redirection-specific)

Things that could go wrong on Day 3 redirection:

- **(F1) Recalibration without substance**: Day 3 가 새 calendar + 4 proposal 만 만들고 OP-0009-Pre / 다른 OP 의 *손에 잡히는 진전* 0. → H8 hazard 직접 발현. 이를 막으려면 R1-b (OP-0009-Pre substantive start) 가 ~1h 이상 *actual mathematical work* 에 도달해야 한다.

- **(F2) Closure-week boundary 깨짐 visibly**: Day 3 가 OP-0009-Pre substantive start 를 working/MF/pre_objective_K_field_tension.md 에 직접 적용하면 (working/ direct edit), W6 가 closure-week 가 아닌 expansion-week 으로 변형된다 (Option B + W6 strategic plan §7 와 모순). Mitigation: Day 3 substantive work 는 *daily-log 02_op_0009_pre_substantive_start.md* 에 daily-log proposal 형태로만; working/ 직접 적용은 W7 D1.

- **(F3) Mathematical work이 안 풀린다**: OP-0009-Pre 의 unordered configuration formalism 은 Tool A2 quotient + Tool A1 stratified space 등의 비-trivial 수학. Day 3 ~1.5h 안에 substantive 진전 안 될 수 있다. → R1 → R2 fallback (design + calendar only).

- **(F4) Pacing inversion**: Day 1 (~10-12h) → Day 2 (~4-5h) → Day 3 (~3-4h, redirection). 점진 taper 가 아니라 plateau (Day 2 ~ Day 3) 가 됨. 검토 후 redirection 이라는 motive 를 고려하면 plateau 는 acceptable; 하지만 Day 4-D7 가 Day 2-D3 와 비슷하게 ~3-4h/day 가 되면 W6 cumulative 가 ~25-28h 로 올라가 W6 strategic plan §3 의 35h target 을 W6 D5-D6 에 close. W6 가 cumulative 면에서 over-deliver.

- **(F5) v1 의 미루어진 작업이 W6 D5-D6 에 다시 수 늘어나면 또 다른 over-delivery**: skeleton refinement (v1 G3.1) + W7 Stage 1 dispatch skeleton (v1 G3.2) 는 W6 D5-D6 reschedule. 만약 D5-D6 가 또 substantive ~4-5h 작업이 되면 W6 cumulative 더 늘어남. Mitigation: D5-D6 plan 작성 시 explicit (D5 = skeleton refinement only 1-1.5h; D6 = skeleton + W7 dispatch skeleton 1-2h; D7 = weekly_summary 만 ~1h).

The hazard tree 는 다음 권고로 수렴: **Day 3 = scope-redirection day, 주로 R1 (full 4-goal substantive) 또는 R2 (triage fallback). v1 administrative work 은 W6 D5-D6 reschedule. W6 cumulative ≤ 22h target (현재 ~17h + Day 3 ~3-4h + D5-D7 ~5-6h).**
